## 📝 Submission Message Template

Use this PR description as a starting point. Replace `[LIST NAME]` and adjust the rationale to fit the specific list.

```markdown
## Add Todoist Playbook

Hi — I'd like to add **Todoist Playbook** to the [LIST NAME] list.

**Repo:** https://github.com/colin-gourlay/todoist-playbook
**Description:** A structured collection of CSV templates, bundled starter kits, AI prompt templates, and GitHub Actions workflows for Todoist power users.

### Why it fits
- Directly addresses productivity and task management — the core theme of this list
- 20+ structured templates covering weekly review, onboarding, sprint ceremonies, SaaS management, and more
- GitHub Actions workflows let users create Todoist projects in one click
- AI prompt templates for generating enriched task content
- Open source under Creative Commons Attribution-ShareAlike 4.0

### Quality signals
- Has a clear README, CONTRIBUTING guide, and CC BY-SA 4.0 licence
- Structured repo layout with per-template READMEs and machine-readable metadata
- CI validates every template on push

Let me know if you'd like any changes to the entry or description. Happy to adjust to match your style guide.
```

## 🔁 Repeatable Process

For each target list:

1. Search GitHub for the list — confirm it is actively maintained
2. Review the list's `CONTRIBUTING.md` for submission rules and format
3. Fork the repo and create a branch: `add-todoist-playbook`
4. Add the entry in the appropriate section, matching the existing style
5. Open a PR using the submission message template above
6. Record the PR URL in the tracking table
7. Check for maintainer feedback after 7 days — respond promptly to any comments
8. On merge: confirm the live listing resolves correctly and update the tracking table
